# IME Live Converter

Windows向けのmacOSライブ変換風IME（日本語入力システム）です。タブやスペースキーを押さずに、入力した文字がリアルタイムで漢字変換されます。

## 機能

- **ローマ字→ひらがな変換**: 入力したローマ字をリアルタイムでひらがなに変換
- **ひらがな→漢字変換（ライブ変換）**: ひらがなを入力中に自動で漢字に変換
- **Viterbi アルゴリズム**: 最適な変換パスを探索して自然な変換結果を生成
- **キーボードフック**: 低レベルキーボードフックで入力をキャプチャ

## ビルド

```bash
cargo build --release
```

## 辞書の準備

本格的な漢字変換には辞書が必要です。IPA辞書（MeCab用）を使用できます。

### 1. IPA辞書のダウンロード

```bash
# 安定ミラー（Debian）からダウンロード
curl -L -o mecab-ipadic.tar.gz \
  "http://deb.debian.org/debian/pool/main/m/mecab-ipadic/mecab-ipadic_2.7.0-20070801+main.orig.tar.gz"

tar xzf mecab-ipadic.tar.gz
```

### 2. バイナリ辞書の生成

```bash
# dict-builder を使って辞書を生成（約39万語、-r/--release 推奨）
cargo run --release -p dict-builder --bin dict-builder -- \
  build ./mecab-ipadic-2.7.0-20070801 ./dictionaries/system.dic
```

`dictionaries/system.dic` があれば CLI 起動時に自動でロードされます
（なければ `dictionaries/sample.dic` にフォールバック）。

## 使い方

### CLI の起動

```bash
cargo run --release -p ime-cli
```

起動すると辞書と学習DB（`ime-learning.db`、自動作成）がロードされます。

### ライブ変換モード（おすすめ）

プロンプトで `:live` と打つとライブ変換モードに入ります。
ローマ字をそのまま打つと、入力停止（250ms）や句読点・「です/ます」などの
文節境界で自動的に仮変換されます。

| キー | 動作 |
|------|------|
| Space | 仮変換の実行 / 次候補 |
| Shift+Space | 前候補 |
| Enter | 確定（学習に記録） |
| Esc | 仮変換をキャンセルしてひらがなに戻す |
| Backspace | 1文字削除（Ctrl+Backspace で全消し） |
| Ctrl+C | ライブモード終了 |

### 全アプリ版（conversion-service）のキー操作

キーの責務を分離しています。**Tab=通常変換／Shift+Space=LLM校正**。

| キー | 動作 |
|------|------|
| **Tab** / ↓ | 通常変換（候補一覧を開く・次候補） |
| ↑ / Shift+Tab | 前候補 |
| 数字 1〜9 | 一覧から番号で直接選択・確定（直近の文節の候補） |
| **Space** | 現在の変換を確定して空白を入れる（非変換中は通常の空白） |
| **Shift+Space** | **LLMで校正**（誤字・文法を修正。下記） |
| Backspace | 直近の変換単語（文節）ごと削除 |
| → | 先頭文節を部分確定 |
| Enter / 句読点 | 確定（学習に記録） |
| 半角/全角 | 変換ON/OFF |
| Ctrl+Space | 変換機能の緊急ON/OFF |

## ローカルLLM校正（任意）

変換中に **Shift+Space** を押すと、統計エンジンの変換結果を「下書き」として
LLM に渡し、**誤字脱字・文法の誤りだけを直して厳密に正しい日本語**に
校正します（例: `ワタシワ学生です` → `私は学生です`）。読みも一緒に
渡すので、言い換えや意味の改変が起きにくい設計です。バックグラウンド
実行で、LLM が無ければ下書き（統計結果）のままなので、未設定でも
通常利用に支障はありません。

### セットアップ（推奨: Docker で一元管理）

ホストにインストールを撒かず、隔離されたコンテナ（サンドボックス）で
LLM を動かします。管理は `docker compose` に一元化されます。

```bash
# リポジトリ直下で
cp .env.example .env          # 使うモデルを変えたい場合は編集
docker compose up -d          # 起動（初回はモデルを自動DL。数分かかることあり）
docker compose logs -f        # 進捗確認
# 停止するとき: docker compose down
```

コンテナが `localhost:11434` で待ち受け、アプリは自動でそこに接続します。
conversion-service 起動時に `LLM: 利用可能 (model=...)` と表示されれば準備完了。
`LLM: 未接続` の場合は Space のLLM選択だけ無効（通常変換は動作）。

> モデルは名前付きボリューム `ollama` に永続化されるので、次回以降は即起動します。
> GPU を使う場合は `docker-compose.yml` の `deploy.resources` のコメントを外してください。

### セットアップ（代替: Ollama を直接インストール）

Docker を使わない場合は Ollama を直接入れても構いません。

```bash
# https://ollama.com からインストール後
ollama pull qwen2.5:3b
ollama serve
```

### 環境変数で挙動を調整（conversion-service 起動前に設定）

| 変数 | 既定 | 説明 |
|------|------|------|
| `IME_LLM_MODEL` | `ime-jp` | 使用するモデル名（日本語特化 Gemma 2 2B の別名） |
| `JP_BASE_MODEL` | `hf.co/alfredplpl/gemma-2-2b-jpn-it-gguf:Q4_K_M` | 取得するベースモデル |
| `IME_LLM_URL` | `http://localhost:11434/api/generate` | 生成APIのURL |
| `IME_LLM_TIMEOUT_MS` | `30000` | タイムアウト(ms。初回はモデル読込で十数秒かかる) |

> **モデルについて**: かな漢字変換は「音を変えずに漢字へ書き換える」特殊なタスクで、
> 多言語モデル（qwen 等）は中国語が混ざるなど精度が出ません。既定では
> **日本語特化の Gemma 2 2B** を使い、「音を変えない書き換え」に厳格化した
> プロンプトを与えることで精度を高めています。それでも小型モデルのため
> 長文では崩れることがあります（短めの区切りで変換すると安定します）。

### 行単位モード（検証用）

プロンプトにそのまま入力すると変換候補一覧が表示されます。

| コマンド | 説明 |
|----------|------|
| `:live` | ライブ変換モード |
| `:commit <N>` | N番目の候補を確定して履歴学習 |
| `:user add <読み> <表記>` | ユーザー辞書に登録 |
| `:typo <入力>` | 誤字補正候補のみ表示 |
| `:nbest <入力> [N]` | Viterbi N-best のみ表示 |
| `:help` / `:quit` | ヘルプ / 終了 |
